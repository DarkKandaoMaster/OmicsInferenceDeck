# 环境配置

OmicsInferenceDeck 是一个前后端分离项目：

- 前端：Nuxt / Vue，默认访问后端 `http://127.0.0.1:8000/api`
- 后端：FastAPI / Python
- 统计分析与部分绘图：后端通过 `Rscript` 调用仓库中的 R 脚本

因此，复现时需要同时准备 Python、R 和 Node.js 环境。下面命令以 Windows PowerShell 为例，默认在仓库根目录执行。

## 1. 准备基础环境

安装以下工具：

- Python 3.11，建议使用 Conda 环境
- R 4.3 或更高版本：<https://cloud.r-project.org/bin/windows/base/>
- Rtools：<https://cloud.r-project.org/bin/windows/Rtools/>
- Node.js，并启用 pnpm（项目锁定 pnpm）

确认 `Rscript` 已加入 PATH：

```powershell
Rscript --version
```

如果提示找不到命令，把 R 安装目录下的 `bin` 加入系统 PATH，例如：

```text
C:\Program Files\R\R-4.4.1\bin
```

## 2. 安装后端 Python 依赖

新建环境：

```powershell
conda create -n omics python=3.11 -y
conda activate omics
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

如果使用已有 Conda 环境，也可以直接用该环境的 Python 安装：

```powershell
& "D:\Programs\DevEnvironments\Anaconda\anaconda3\envs\myenv\python.exe" -m pip install -r requirements.txt
```

Python 依赖以 [requirements.txt](requirements.txt) 为准。

## 3. 安装 R 依赖

在仓库根目录运行：

```powershell
Rscript install_r_packages.R
```

脚本会自动跳过已安装的包，并安装项目需要的 CRAN、Bioconductor 和 GitHub 包。GitHub 源码包（如 NEMO、PintMF、MOSD）需要 Rtools 能正常工作。

## 4. 安装前端依赖

```powershell
cd frontend
corepack enable
pnpm install
cd ..
```

前端依赖以 [frontend/package.json](frontend/package.json) 和 [frontend/pnpm-lock.yaml](frontend/pnpm-lock.yaml) 为准。

## 5. 启动项目

先启动后端。由于当前后端代码使用了相对于 `backend` 目录的导入方式，建议进入 `backend` 后启动：

```powershell
conda activate omics
cd backend
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

后端文档地址：

```text
http://127.0.0.1:8000/docs
```

再打开另一个 PowerShell，启动前端：

```powershell
cd frontend
pnpm dev
```

前端默认地址：

```text
http://127.0.0.1:3000
```

## 6. 快速验证

Python 依赖：

```powershell
python -c "import fastapi, pandas, sklearn, umap, lifelines, pyrea, fitz; print('Python OK')"
```

R 依赖：

```powershell
Rscript -e "pkgs <- c('arrow','limma','edgeR','clusterProfiler','SNFtool','ggplot2'); stopifnot(all(vapply(pkgs, requireNamespace, logical(1), quietly = TRUE))); cat('R OK\n')"
```

前端：

```powershell
cd frontend
pnpm dev
```

浏览器能打开 `http://127.0.0.1:3000`，并且后端 `http://127.0.0.1:8000/docs` 可访问，即环境配置完成。

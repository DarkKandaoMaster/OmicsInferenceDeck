# 环境配置

OmicsInferenceDeck 是一个前后端分离项目：

- 前端：Nuxt / Vue，默认访问后端 `http://127.0.0.1:8000/`
- 后端：FastAPI / Python
- 统计分析与部分绘图：后端通过 `Rscript` 调用仓库中的 R 脚本

因此，复现时需要同时准备 Python、R 和 Node.js 环境。下面命令以 Windows PowerShell 为例，默认在仓库根目录执行。

## 1. 准备基础环境

安装以下工具：

- Python 3.11，建议使用 Conda 环境
- R 4.3 或更高版本：<https://cloud.r-project.org/bin/windows/base/>
- Rtools：<https://cloud.r-project.org/bin/windows/Rtools/>
- Node.js，并启用 pnpm（项目锁定 pnpm）

## 2. 安装后端 Python 依赖

新建环境：

```powershell
conda create -n omics python=3.11 -y
conda activate omics
pip install -r requirements.txt
```

Python 依赖以 [requirements.txt](requirements.txt) 为准。

## 3. 安装 R 依赖

在仓库根目录运行：

```powershell
Rscript install_r_packages.R
```

脚本会自动跳过已安装的包，并安装项目需要的 CRAN、Bioconductor 和 GitHub 包。GitHub 源码包（如 NEMO、PintMF、MOSD）需要 Rtools 能正常工作。

## 4. 安装前端依赖

```powershell
cd frontend
pnpm install
```

前端依赖以 [frontend/package.json](frontend/package.json) 和 [frontend/pnpm-lock.yaml](frontend/pnpm-lock.yaml) 为准。

## 5. 启动项目

先启动后端。由于当前后端代码使用了相对于 `backend` 目录的导入方式，所以需要进入 `backend` 后启动：

```powershell
conda activate omics
cd backend
python main.py
```

再打开另一个 PowerShell，启动前端：

```powershell
cd frontend
pnpm dev
```
